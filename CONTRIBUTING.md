# Contributing

## Правила разработки

- Используйте отдельные ветки для задач: `feature/*`, `fix/*`, `docs/*`.
- Перед коммитом проверяйте код и миграции локально.
- Не коммитьте секреты, `.env`, локальные базы и служебные артефакты.
- Для изменений моделей добавляйте корректные миграции.
- Для функциональных изменений обновляйте документацию в `README.md`.

## Стиль коммитов

- Короткий и понятный заголовок на английском.
- Один коммит = одна логическая группа изменений.

## Pull Request

- Кратко опишите цель, что изменено и как проверить.
- Приложите скриншоты интерфейса при изменениях в UI.
